# Настройка кодировки консоли Windows

## Проблема
При запуске приложения в консоли Windows могут отображаться "кракозябры" вместо русских букв:
```
[MAIN] ╨Ю╨║╨╜╨╛ ╤Г╨╜╨╕╤З╤В╨╛╨╢╨╡╨╜╨╛
```

## Решение (автоматическое)
Приложение автоматически устанавливает кодировку UTF-8 (code page 65001) при запуске на Windows.

## Ручная настройка (если автоматическая не сработала)

### Способ 1: Через команду chcp
Перед запуском приложения выполните в PowerShell:
```powershell
chcp 65001
npm run dev
```

### Способ 2: Постоянная настройка PowerShell
Добавьте в профиль PowerShell (`$PROFILE`):
```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
chcp 65001 | Out-Null
```

Чтобы открыть профиль:
```powershell
notepad $PROFILE
```

### Способ 3: Настройка через реестр (для всех приложений)
1. Откройте Regedit
2. Перейдите к `HKEY_CURRENT_USER\Console`
3. Создайте DWORD параметр `CodePage` со значением `65001` (десятичная система)
4. Перезапустите терминал

## Проверка кодировки
```powershell
chcp
```
Должно показать: `Active code page: 65001`

## Примечание
- Исправление работает только для консольных логов Electron (main process)
- Логи renderer process (Vue компоненты) выводятся в DevTools и не имеют проблем с кодировкой
